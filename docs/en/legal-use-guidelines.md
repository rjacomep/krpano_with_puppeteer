# Legal & Ethical Use Guidelines - English (UK)

[Back to main documentation](./README.md)

## Purpose & Legal Status

**Krpano QA Tester** is a legitimate quality assurance and automation framework designed for testing panoramic tours you own or have explicit authorisation to test.

✅ **LEGAL USES:**
- QA testing of your own krpano tours
- Testing tours you have explicit permission to test
- Integration into CI/CD pipelines for automated QA
- Visual regression testing between versions
- Performance and compatibility analysis
- Accessibility validation
- Internal documentation and reporting

❌ **ILLEGAL/UNAUTHORISED USES:**
- Testing tours without authorisation
- Circumventing access controls
- Unauthorised content redistribution
- Scraping copyrighted content
- Commercial resale of extracted assets
- Using tours for competitive analysis without permission

---

## Authorisation Requirements

### Before Using This Tool:

1. **Verify Ownership or Permission**
   - You own the tour being tested, OR
   - You have written authorisation from the tour owner

2. **Respect Terms of Service**
   - Check the website's terms of service
   - Honour any robots.txt restrictions
   - Follow rate-limiting guidelines

3. **Respect Intellectual Property**
   - Do not redistribute extracted content
   - Do not use content for commercial purposes without permission
   - Credit creators appropriately

4. **Use Internal Testing Only**
   - Testing results are for your internal QA team
   - Do not share extracted content publicly
   - Keep test environment private

---

## Technical Compliance

### Respectful Testing Practices

This framework includes built-in respectful practices:

- **Throttling**: Configurable delays prevent server overload
- **Rate Limiting**: Exponential backoff for failures
- **User Agent Rotation**: Identifies as browser
- **Concurrency Control**: Limits parallel requests

## Data Privacy & Security

### What This Tool Does:
✅ Creates local test environment mirrors
✅ Generates QA screenshot reports
✅ Logs test performance metrics

### What This Tool Does NOT Do:
❌ Steal or extract personal data
❌ Circumvent authentication
❌ Modify or corrupt data
❌ Bypass security measures

## Acceptable Use Policy

Users agree to:

1. **Use Authorisation**
   - Only test tours you own or have permission for
   - Verify authorisation before testing
   - Maintain records of authorisation

2. **Respectful Operation**
   - Honour configured delays and rate limits
   - Do not attempt to bypass throttling
   - Do not make excessive requests

3. **Legal Compliance**
   - Follow local and international laws
   - Respect copyright and intellectual property
   - Honour website terms of service

4. **Responsible Disclosure**
   - If you find security issues, report them responsibly
   - Do not publicly disclose vulnerabilities
   - Contact website owner directly

5. **No Redistribution**
   - Do not share extracted content publicly
   - Do not use for commercial purposes without permission
   - Keep test environments private

---

## Summary: The Rule

**Use this tool ONLY for testing tours you own or have explicit permission to test.**

If you wouldn't email the tour owner and ask "Is it OK if I test your tour with this tool?", don't do it.
